# video_utils

CLI-утилита на Go для склейки списка видеофайлов из YAML-конфига в один файл через
`ffmpeg`. Разнородные источники (разные кодеки, разрешение, fps) автоматически
приводятся к общему формату. Перед роликами с заголовком вставляется текстовая
заставка. Отдельные ролики можно ускорить под заданную длительность и наложить на них
исходный таймкод от начала видео.

## Возможности

- Чтение списка видео и опций из YAML-конфига.
- Текстовая заставка (чёрный фон, белый текст) перед роликами, у которых задан `name`.
- Автоматический выбор «оптимального» формата вывода по входным файлам:
  - кодеки — максимально совместимые среди встретившихся (приоритет H.264 / AAC, MP4);
  - разрешение и fps — максимальные среди входных файлов;
  - меньшие ролики масштабируются с сохранением пропорций (letterbox/pad).
- Корректная склейка разнородных источников: каждый сегмент приводится к общему
  формату по отдельности, затем сегменты быстро склеиваются без перекодирования
  (concat-демультиплексор, потоковое копирование). Ролики без аудио дополняются тишиной.
- Ускорение ролика под заданное время (`fit_duration`): видео ускоряется так, чтобы
  полностью уложиться в указанную длительность (`MM:SS` или `HH:MM:SS`). При ускорении
  исходное аудио заменяется тишиной. Опция `show_original_time` накладывает исходное время
  от начала ролика (`HH:MM:SS`, идущее в оригинальном темпе) в левом верхнем углу.
- Параллельное кодирование сегментов с ограничением по числу одновременных задач
  (по умолчанию CPU/2). Порядок склейки сохраняется. Управление: флаг `-j` или
  опция `jobs`; при `-j 1` — последовательная обработка.
- Понятные логи: вывод ffmpeg скрыт, показывается только текущий обрабатываемый
  файл (его `name`) и время его обработки. Ошибки выделяются красным,
  предупреждения — зелёным (отключить цвет: переменная окружения `NO_COLOR`).

## Требования

- Go 1.25+
- `ffmpeg` и `ffprobe` в `PATH`

## Сборка

```sh
go build -o video_utils .
```

## Использование

```sh
video_utils -c config.yaml
```

Флаги:

| Флаг        | Описание                                            |
|-------------|-----------------------------------------------------|
| `-c`        | путь к YAML-конфигу (обязательно)                    |
| `-o`        | переопределить путь к итоговому файлу                |
| `-j`        | число одновременно кодируемых сегментов (0 = авто: CPU/2) |
| `--dry-run` | показать команды ffmpeg и выйти (без кодирования)   |
| `-v`        | подробный вывод (параметры проб и целевого формата) |

## Формат конфига

См. [`config.example.yaml`](config.example.yaml).

```yaml
output: result.mp4
options:
  splash_seconds: 3      # длительность заставки, сек (по умолчанию 2)
  jobs: 4                 # параллельных задач кодирования (0/нет = авто: CPU/2)
  temp_dir: "D:/tmp/vc"   # каталог для временных файлов (по умолчанию системный)
  font: "C:/Windows/Fonts/arial.ttf"
  background: black
  text_color: white
videos:
  - name: "Часть 1"      # есть name -> заставка перед видео
    path: "C:/video/1.mp4"
  - path: "C:/video/2.mp4"   # нет name -> без заставки
    fit_duration: "00:30"    # ускорить до 30с (MM:SS или HH:MM:SS); при ускорении звук -> тишина
    show_original_time: true # показать исходное время от начала ролика (HH:MM:SS, слева сверху)
```

Ключи `fit_duration` и `show_original_time` задаются у каждого видео независимо.
Если `fit_duration` не меньше исходной длительности ролика, ускорение не применяется.
